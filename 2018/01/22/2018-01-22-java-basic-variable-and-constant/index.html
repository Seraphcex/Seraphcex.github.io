Pratice

Identifier

It’s a symbol that we use to name variable,class,method etc.

When we use identifier,there are some rules need to be followed:

  1. A identifier can consists of letter,number,_,$,and can’t starts with number.
  2. Identifier can’t be Java keyword or reserved word.
  3. Identifier is case sensitive.
  4. Naming a identifier better can inform its meaning or purpose.

Variable

We can see it as a box,we can put anything we want into the box,and replace it with new item at any time.

In Java,we use 3 elements to describe variable:type,name and value.

Also we need to follow rules like:

  1. A variable name can consists of letter,number,_,$,and can’t starts with number.
  2. Variable name can’t be Java keyword or reserved word.
  3. Variable name is case sensitive.
  4. Naming a variable name better can inform its meaning or purpose.

In order to form a better program habit,there are several naming methods:

  1. Hungarian notation
  2. Camel case
  3. Pascal case

Practice

Rules of using variables

  1. Variables must declare before using it.

  2. You can declare and initialize variables in the same time.

Also you can first declareit,and then assign it afterward.

  1. You can only assign one value to a variable,but you can change multiple time.

  2. The variables that you have declared in method main must be assigned first,and then you can output it.

  3. Although you might not have error shows up,but in real life development,I suggest only using english when you naming variables.It’s easier to have safety issues if you use other languages,such as having gibberish when you are working on cross platform.

Constant

We see it as a kind of special variable,when its value has been assigned,it doesn’t allow to change during the operation of your program.

  • syntax: final type name = value;

Using constant in your program can improve maintainability,and notice we normally use capital letter while naming a constant.
